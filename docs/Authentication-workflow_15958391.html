Access token states

The access token can now have 3 states:

  • active - the access token has not been revoked, and is not expired

  • inactive - the access token cannot be used anymore. It has been revoked and will be deleted soon

Tokens lifecycles

Access tokens

Access tokens are JWT with these properties:

  • sub the subject of the token (the user ID)

  • iat the issued date of the token

  • exp the expiration date of the token

The access tokens are stored in the database. When a token is verified, the JWT is validated & the token is compared with the one stored in database.

Refresh tokens

Refresh tokens are JWT with these properties:

  • sub the subject of the token (the access token this refresh token refers to)

  • iat the issued date of the token

  • exp the expiration date of the token

The refresh tokens are stored in the database. When a token is verified, the JWT is validated & the token is compared with the one stored in database.

The expired refresh tokens are deleted from the database too thanks to an internal routine.
