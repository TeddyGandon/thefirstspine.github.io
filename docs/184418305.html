Yield allows you to create and play games on the TFS platform. Here's some concepts to understand before going further. To use your app with TFS you should fist bind your users with some Yield users. You can create users in Yield and identify them with a custom ID. Once your users created, you should then create a game on Yield.

Endpoints documentation

This part describes the endpoints of the API.

GET /api/key

Get information about the used key.

Authorization

The X-API-Key header is required with the API Key provided by the Yield team.

Responses

Response code

Description

Example

403 Unauthorized

Invalid API Key

{
  "status": false,
  "error": "Invalid API Key"
}

200 Ok

Normal response

{
  "data": {
    "email": "email@email.com"
  }
}

POST /users

Create a user. You can chose the User ID (under custom_id) to help you bind this user to your app.

Authorization

The X-API-Key header is required with the API Key provided by the Yield team.

Required ?
